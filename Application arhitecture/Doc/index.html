Introduction

This Web application allows to observe the evolution of the digital cats as well as the owners.You can visualize the most important owners,newest "cats" and their family tree.

General design

  • This is a single-page application (SPA),that interacts with the user by dynamically rewriting the current page rather than loading entire new pages from a server.
  • Provides various options structured as follows:
    • A user profile page where the information about a specific user is displayed
    • A cat profile page where the information about a specific cat is displayed
    • A page with the newest cats
    • A page with top owners depending on the number of the cats the own
    • A home page from where you can go to the generate svg/png page to get some statistics from the website

Intermediate steps

Between these two components, some thinking about the spa structure, data handeling from the public API, and data storage on the client side was involved.

Data modeling

This application doesn't have a database to store the informations, we make API requests directly to the official site servers.To work with the data from these requests, we map the json we get after the request to a javascript object.

Storage solutions available at the client level

All the kitties and owners are saved into lists after a user checks their profile. Before an api request, a search through these lists takes place, and an api request is made only if necessary.
